# AI Coding Companion - 进度总结与路线图

## 📊 当前进度总结 (v0.1.0 MVP)

### ✅ 已完成的核心功能

#### 1. **MVP 基础架构**
- [x] 策略模式设计架构 (`src/strategies/`)
- [x] 抽象基类 `MonitorStrategy`
- [x] `LogFileMonitor` - Aider 日志文件监控
- [x] `CompletionDetector` - 完成状态检测器
- [x] `Notifier` - 多通道通知发送器
- [x] 主程序循环和资源管理

#### 2. **Aider 日志监控 (MVP 核心)**
- [x] 监控 Aider 日志文件 (`~/.aider/` 目录)
- [x] 自动检测日志文件路径 (`src/aider_detector.py`)
- [x] 支持多种日志格式：
  - `.aider.chat.history.md` (隐藏文件，新版本)
  - `.aider.history` (隐藏文件，旧版本)
  - `chat.history.md` (普通文件，新版本)
  - `history` (普通文件，旧版本)
  - `*.history.md` (带时间戳版本)
- [x] 智能完成标记检测：
  - `"Added"` - 文件添加到对话
  - `"Committed"` - 提交代码更改
  - `"^>"` - 返回提示符 (正则表达式)

#### 3. **通知系统**
- [x] 飞书通知支持 (主要渠道)
- [x] Discord、邮件、Telegram 接口预留
- [x] OpenClaw 工具集成支持
- [x] 格式化通知消息

#### 4. **配置管理**
- [x] YAML 配置文件支持
- [x] 自动检测与手动配置结合
- [x] 多项目监控支持
- [x] 灵活的完成标记配置

#### 5. **代码质量**
- [x] 完善的错误处理
- [x] 详细的日志输出
- [x] 模块化设计
- [x] 完整的测试套件
- [x] 编码问题处理 (Windows 兼容)

#### 6. **权限检测系统 (新增)**
- [x] `PermissionDetector` - 权限检测器
- [x] 支持 Aider 各种权限提示格式：
  - `Add file to the chat? (Y)es/(N)o/(D)on't ask again [Yes]:`
  - `Create new file? (Y)es/(N)o [Yes]:`
  - `Apply these changes? (Y)es/(N)o:`
  - `是否执行这个编辑？(y/n)` (中文)
- [x] 四种处理模式：
  - `ask` - 询问用户（发送通知）
  - `auto` - 根据白名单自动决策
  - `auto_approve` - 自动批准所有请求（最宽松）
  - `silent` - 静默拒绝所有请求
- [x] 白名单系统（正则表达式匹配文件路径）
- [x] 实时通知集成
- [x] 避免重复通知（60秒冷却时间）
- [x] 完整的测试套件

### 🎯 MVP 使用场景验证

**场景**: 用户使用 Aider 进行编程，无需一直盯着屏幕
```
用户: "帮我修复这个bug"
    ↓
Aider 开始工作 (可能需要几分钟)
    ↓
用户去做其他事情 ✓
    ↓
Aider 完成 → 自动检测 → 发送飞书通知
    ↓
用户回来继续下一轮工作
```

## 🚀 下一步：一体化启动器实现方案

### 目标
实现用户只需启动一个程序，即可：
1. 自动检测电脑中的AI编程工具
2. 交互式选择工具和配置
3. 自动启动选定的工具
4. 全程监控工具输出
5. 按规定发送提醒信息

### 实现方案

#### 1. **一体化启动器架构**
```
companion_launcher.py
├── 工具检测模块 (ToolDetector)
├── 交互式设置向导 (InteractiveSetup)
├── 工具启动器 (ToolLauncher)
├── 监控管理器 (MonitorManager)
└── 配置生成器 (ConfigGenerator)
```

#### 2. **核心功能模块**

##### 2.1 智能工具检测器
```python
class SmartToolDetector:
    def detect_installed_tools() -> List[ToolInfo]
    def get_launch_command(tool_id, project_path) -> List[str]
    def check_tool_availability(tool_id) -> bool
```

##### 2.2 交互式配置向导
```python
class ConfigWizard:
    def run_interactive_setup() -> Config
    def select_tool(tools: List[ToolInfo]) -> ToolInfo
    def select_project_path() -> str
    def select_permission_mode() -> str
    def generate_config() -> Dict
```

##### 2.3 一体化启动管理器
```python
class CompanionLauncher:
    def launch_tool(tool_id, project_path) -> subprocess.Popen
    def start_monitoring(config_path) -> Monitor
    def handle_user_interaction() -> None
    def cleanup_resources() -> None
```

#### 3. **用户工作流程**
```
用户运行: python companion_launcher.py
         ↓
自动检测可用AI工具
         ↓
显示工具列表供选择
         ↓
选择项目路径和批准模式
         ↓
生成配置文件
         ↓
自动启动选定的AI工具
         ↓
启动监控进程
         ↓
[用户与AI工具交互]
         ↓
监控器检测完成状态
         ↓
发送通知提醒
```

#### 4. **关键技术实现**

##### 4.1 工具自动启动
- **命令行工具** (Aider): 使用 `subprocess.Popen` 启动
- **GUI应用** (Cursor): 使用系统命令打开 (`start`, `open`, `xdg-open`)
- **VS Code**: 使用 `code` 命令打开项目

##### 4.2 监控集成
- 复用现有的 `TerminalMonitor` 监控终端输出
- 对于GUI应用，探索进程监控或日志监控方案
- 统一的完成检测和通知发送

##### 4.3 配置管理
- 自动生成用户特定的配置文件
- 支持配置保存和加载
- 提供配置编辑界面
