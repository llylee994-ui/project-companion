# 权限检测功能文档

## 概述

AI Coding Companion 现在支持检测 Aider 的权限询问，并可以实时通知用户或自动处理这些请求。当 Aider 询问"是否执行这个编辑？(y/n)"时，系统可以立即通知用户，而不是等待整个会话结束。

## 功能特性

### 1. 权限检测
- 检测 Aider 的中英文权限询问
- 支持多种权限类型：文件编辑、文件创建、文件删除、Git 提交、Git 推送
- 自动提取涉及的文件路径
- 支持正则表达式模式匹配

### 2. 实时通知
- 检测到权限请求时立即发送通知
- 避免重复通知（60秒冷却时间）
- 通知包含：项目名称、权限类型、涉及文件、决策状态

### 3. 自动决策
- 三种处理模式：询问(ask)、自动(auto)、静默(silent)
- 基于白名单的自动决策系统
- 支持正则表达式模式匹配文件路径

### 4. 白名单系统
- 可配置的文件路径模式
- 支持正则表达式
- 示例：允许所有 `.txt` 文件、允许测试文件等

## 配置说明

### 权限配置示例

```yaml
# config.yaml
permissions:
  mode: "ask"             # ask（询问）/ auto（自动）/ silent（静默）
  auto_allow_patterns:
    - ".*\\.md$"           # 允许所有 Markdown 文件
    - ".*\\.txt$"          # 允许所有文本文件
    - ".*test.*\\.py$"     # 允许测试文件
    - ".*docs?/.*"         # 允许文档目录
    - ".*config/.*"        # 允许配置目录
```

### 模式说明

1. **ask 模式（默认）**
   - 检测到权限请求时发送通知
   - 用户需要在 Aider 中手动回复 y/n
   - 适合重要项目或需要人工审核的情况

2. **auto 模式**
   - 根据白名单自动决策
   - 匹配白名单：自动允许
   - 不匹配白名单：自动拒绝
   - 不发送通知（除非需要用户确认）

3. **silent 模式**
   - 自动拒绝所有权限请求
   - 不发送任何通知
   - 适合只读监控场景

## 支持的权限提示模式

### 中文提示
- `是否执行这个编辑？(y/n)`
- `是否应用这些更改？(y/n)`
- `是否创建文件？(y/n)`
- `是否删除文件？(y/n)`
- `是否提交更改？(y/n)`
- `是否推送更改？(y/n)`

### 英文提示
- `Apply these changes? (y/n)`
- `Make these changes? (y/n)`
- `Do you want to apply this edit? (y/n)`
- `Create this file? (y/n)`
- `Delete this file? (y/n)`
- `Commit these changes? (y/n)`
- `Push these changes? (y/n)`

## 文件路径提取

系统会自动从权限提示中提取文件路径，支持以下格式：
- `文件: /path/to/file.txt`
- `file: /path/to/file.txt`
- `创建: /path/to/file.txt`
- `create: /path/to/file.txt`
- `编辑: /path/to/file.txt`
- `edit: /path/to/file.txt`
- `删除: /path/to/file.txt`
- `delete: /path/to/file.txt`

## 使用场景

### 场景1：开发环境
```yaml
permissions:
  mode: "auto"
  auto_allow_patterns:
    - ".*\\.md$"           # 允许文档修改
    - ".*test.*\\.py$"     # 允许测试文件
    - ".*\\.txt$"         # 允许文本文件
    - ".*docs?/.*"        # 允许文档目录
```

### 场景2：生产环境
```yaml
permissions:
  mode: "ask"             # 重要项目，总是询问
  auto_allow_patterns: [] # 无自动权限
```

### 场景3：只读监控
```yaml
permissions:
  mode: "silent"          # 静默拒绝所有权限
  auto_allow_patterns: [] # 无自动权限
```

## 技术实现

### 核心组件

1. **PermissionDetector 类**
   - 权限检测和类型识别
   - 文件路径提取
   - 决策逻辑实现

2. **集成到主监控循环**
   - 在 `check_project` 方法中添加权限检测
   - 实时处理权限请求
   - 避免重复通知

3. **通知扩展**
   - 扩展 `Notifier` 类支持权限通知
   - 格式化权限通知消息

### 代码结构

```
src/
├── permission_detector.py    # 权限检测核心逻辑
├── notifier.py              # 扩展的通知功能
└── strategies/log_monitor.py # 日志监控（不变）

main.py                      # 集成权限检测到主循环
```

## 故障排除

### 常见问题

1. **权限未被检测到**
   - 检查日志格式是否匹配
   - 验证正则表达式模式
   - 检查文件编码

2. **重复通知**
   - 系统有60秒冷却时间
   - 检查权限提示是否重复出现
   - 验证通知配置

3. **自动决策不符合预期**
   - 检查白名单模式
   - 验证文件路径匹配
   - 检查模式顺序
